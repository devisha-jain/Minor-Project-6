SQL Minor Project: Bike Store & Airbnb Analysis

A relational SQL project built to practice core querying fundamentals — JOINs, GROUP BY, aggregate functions, and WHERE/HAVING filtering — without relying on subqueries or CTEs. The project has two independent database scenarios, each fully self-contained with its own schema, sample data, and analytical queries.

Contents

FileDescriptionBikeStore_Analysis.sqlRetail chain database — stores, products, brands, customers, orders, and inventoryAirbnb_Analysis.sqlHospitality database — hosts, properties, guests, and bookings

Tech Stack


Database: MySQL (tested in MySQL Workbench)
Concepts used: INNER JOIN, GROUP BY, HAVING, aggregate functions (SUM, COUNT, AVG), CASE WHEN, ORDER BY
Intentionally excluded: subqueries, CTEs — to keep the focus on core relational logic


How to Run

Each .sql file is fully self-contained — it creates its own database, tables, and sample data before running the analytical queries. No external CSVs or setup needed.


Open the .sql file in MySQL Workbench (or any MySQL client)
Select the entire file (Ctrl+A)
Execute the whole script (Ctrl+Shift+Enter, or the "execute script" icon)
This will:

Create the database (BikeStoreProject or AirbnbProject)
Create all tables with proper foreign key relationships
Insert sample data (~20–30 rows per major table)
Run all 5 analytical queries, each shown in its own result tab





To re-run just one query later, make sure you first run USE BikeStoreProject; (or USE AirbnbProject;) in that session, then execute the specific query.

Notes


All queries use column aliases (AS) for readable output.
ORDER BY is applied to every final result set for consistent, readable reporting.
Sample data is fictional but designed to produce realistic, non-trivial query results (ties, zero-stock cases, mixed nationalities, etc.).
